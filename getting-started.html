Getting Started with Water Framework

Introduction

Welcome to Water Framework! This guide will help you get started with our framework and show you how to build your first application.

Prerequisites

Before you begin, make sure you have the following tools installed on your system:

  • Java 17 or higher - Required for running Water Framework applications
  • Gradle - For building and managing your project dependencies
  • Node.js - Required for the Water Generator tool
  • Yeoman - For scaffolding new Water Framework projects

To install Yeoman and the Water Generator, run the following commands:


npm install -g yo
npm install -g generator-water --registry https://nexus.acsoftware.it/nexus/repository/npm-acs-public-repo

Creating Your First Project

To create a new Water Framework project, follow these steps:

1. Generate a New Project

First, make sure you're in your workspace folder (not inside any project folder). Then run the following command:


yo water:new-project

The generator will prompt you for some basic information about your project:

  • Project name
  • Base package name
  • Framework selection (Spring, OSGi, Quarkus, etc.)
  • Additional modules to include

2. Project Structure

After generation, your project will follow a modular structure with the following components:


your-project/
├── your-project-api/           # API interfaces and models
├── your-project-model/         # Data models and entities
├── your-project-service/       # Core business logic
├── your-project-service-integration/  # Integration layer
├── your-project-service-spring/       # Spring-specific implementation
├── build.gradle               # Main build file
├── settings.gradle           # Project settings
└── gradle.properties         # Gradle properties

Module Organization

  • API Module: Contains interfaces and contracts that define your module's public API
  • Model Module: Contains data models, entities, and DTOs
  • Service Module: Implements the core business logic
  • Service Integration Module: Handles integration with other services and external systems
  • Service Spring Module: Provides Spring-specific implementations and REST endpoints

REST Compatibility Note

For REST compatibility, the generator creates a separate Spring service project. This is necessary because Spring does not automatically support JAX-RS annotations. The Spring service module will handle:

  • REST endpoint implementations
  • Spring-specific configurations
  • JAX-RS to Spring REST mapping

3. Running Your Project

There are several ways to run your Water Framework project:

Option 1: Run as Spring Application

To run the project as a standalone Spring application:


./gradlew :your-project-service-spring:run

Option 2: Deploy to OSGi Container

To deploy your module to an OSGi container like Karaf:

  1. Navigate to your workspace folder (where all your projects are located)
  2. Build the project using the Water Generator: 
    
yo water:build
  3. Copy the generated bundles to your OSGi container's deploy directory
  4. Start your OSGi container

Option 3: Run Tests

To run the generated tests using the Water Generator:

Make sure you're in your workspace folder, then run:


yo water:build --withTests

For more detailed test reports:


yo water:build --withTests --info
